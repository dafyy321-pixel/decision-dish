# 《等会吃啥》营销方案
## 专为二本AI专业大三学生定制的校园产品冷启动方案

---

## 📋 方案概览

|| 项目 | 内容 |
||------|------|
|| **产品名称** | 等会吃啥 (Decision Dish) |
|| **目标人群** | 本校8000名学生（女生偏多） |
|| **核心目标** | 7天内获得50个真实用户尝试 + 长期积累用户与变现 |
|| **推广预算** | 500元 |
|| **时间投入** | 每天2-3小时 |
|| **团队配置** | 1人（AI专业大三学生） |
|| **校内资源** | 校园论坛/表白墙/树洞等线上平台 |
|| **竞争环境** | 无直接竞品 |

### 执行摘要（Executive Summary）
- 市场：本校约8000名学生，女生偏多；饭点前“吃啥”高频刚需。
- 目标：7天完成50个有效尝试（≥1次抽取），CAC ≤ 10元，次日留存 ≥ 30%，分享率 ≥ 15%。
- 策略：种子用户 → 食堂地推 → 表白墙/QQ空间/KOL → 活动裂变与社群沉淀。
- 资源：预算500元；每日2-3小时执行；以宿舍为传播单元（1带4）。

### OKR（7天冷启动）
- O：完成冷启动并验证增长闭环可行。
- KR1：新增≥50名有效用户（当周至少抽取1次）。
- KR2：次日留存率≥30%（D1）；周留存≥20%（W1）。
- KR3：分享率≥15%（使用后触发分享/转发行为）。
- KR4：访问→使用转化率≥60%（打开即抽取）。
- KR5：单次新增成本CAC≤10元，预算控制在500元内。

### 漏斗目标值（Week 1）
- 曝光：≥1200（线下触达+线上帖子/KOL）。
- 访问UV：≥80-100（短链/二维码统计）。
- 首次抽取：≥50-60（CVR≥60%）。
- 次日留存：≥15-20（D1 30%-35%）。
- 分享人数：≥8-10（分享率≥15%）。

### 预算拆解与ROI 预估（Week 1）
- 食堂地推 100元：触达50-80人→使用15-25人，CAC≈4-7元。
- 公众号/QQ空间/KOL 100元：触达2000-5000→使用8-15人，CAC≈6-12元。
- 活动福利 200元：裂变/留存激励（前20名限定），目标新增20人，成本≤10元/人。
- 应急 100元：补贴地推/紧急投放。
- 目标ROI（短期估算）：若第2月拉到3家广告主×300元/月≈900元/月，则Week1投入回本周期≈1-2月。

### 黄金时段（执行节奏）
- 11:20-12:40、17:20-18:40：食堂/校门口拦截转化。
- 20:00-21:30：宿舍群传播与私域维护。
- 22:00-23:00：数据复盘+次日物料准备。

## 🎯 第一部分：核心策略定位

### 1.1 产品定位（针对女生为主的校园市场）

**Slogan候选：**
1. "今天吃啥不纠结，转个盘就搞定！" ✨（推荐）
2. "选择困难症救星，一秒决定今天吃啥"
3. "姐妹们别纠结了，让转盘帮你选！"

**核心卖点（3秒电梯演讲）：**
> "再也不用在宿舍门口纠结半小时吃啥了！打开转个盘，1秒出结果，还能自定义你爱吃的店，已经帮XX个同学解决了选择困难症！"

### 1.2 用户画像分析

**主要目标用户（女生为主）：**

| 用户类型 | 占比 | 痛点 | 使用场景 | 触达策略 |
|---------|------|------|---------|---------|
| **选择困难症女生** | 40% | 每次吃饭都要纠结很久，询问舍友意见 | 中午/晚上饭点前 | 宿舍楼投放、闺蜜裂变 |
| **宿舍集体决策** | 30% | 4个人4个意见，难统一 | 集体聚餐、周末出去玩 | 宿舍长推广、群裂变 |
| **单身男生** | 20% | 想尝试新店但懒得选 | 日常三餐 | 游戏/技术社群 |
| **情侣** | 10% | 约会时不知道吃啥，互相推脱 | 周末约会 | 表白墙场景化推广 |

**核心洞察：**
- 女生更容易分享有趣的小工具（口碑传播强）
- 宿舍是天然的传播单元（1带4效应）
- 饭点前15-30分钟是使用高峰期（11:30、17:30）

### 1.3 竞争优势（为什么选你不选别的）

| 对比对象 | 他们的问题 | 我们的优势 |
|---------|-----------|-----------|
| 微信群问"吃啥" | 没人回复/意见不统一 | ✅ 1秒出结果，公平公正 |
| 美团/饿了么搜索 | 选择太多更纠结 | ✅ 只推荐20家精选，降低决策成本 |
| 自己脑子想 | 总想不出来，或总是同一家 | ✅ 随机推荐，帮你发现新店 |
| 线下转盘/抽签 | 太麻烦，没带工具 | ✅ 手机打开就能用，随时随地 |

### Day 3-4：验证期（目标：累计30人，新增20人）

**核心任务：线下精准触达 + 线上社群扩散**

#### ✅ 线下地推（最有效！预算200元）

**策略：精准小范围地推 > 大规模盲目推广**

**场景1：食堂门口/饭点拦截（最推荐）**

**时间：** 中午11:30-12:30，晚上17:30-18:30
**地点：** 食堂门口/校门口餐饮集中区
**人数：** 建议找1-2个朋友帮忙（付50-100元/天）

**话术流程：**
```
1. 拦截："同学，请问下你们平时吃饭会纠结吃啥吗？"
   （对方如果说"会"，继续；说"不会"，礼貌放行）

2. 引导："我做了个小工具，1秒帮你决定，能帮忙试试吗？很快的"
   （递上手机/让对方扫码）

3. 演示："你看，转一下就出结果了～还能加自己喜欢的店"
   （让对方自己操作一次）

4. 裂变："觉得有用的话可以分享给舍友哦，你们宿舍集体用更好玩～"
   （引导现场分享到宿舍群）

5. 激励："扫码加我微信，拉你进用户群，定期有活动～"
   （建立私域流量）
```

**物料准备：**
- [ ] 打印50张小卡片（预算50元）
  - 正面：产品名称 + Slogan + 二维码
  - 背面：加微信送"神秘福利"（后续可推广新功能）
- [ ] 准备小礼品：棒棒糖/便签纸（预算50元，5元/包×10包）
  - 话术："试用下就送小礼物，不占用你时间～"

**预期效果：**
- 2小时触达：50-80人
- 转化率：30%（15-25人真实使用）
- 裂变率：20%（3-5人分享到宿舍群）

**场景2：宿舍楼扫楼（女生宿舍优先）**

⚠️ **注意：** 你是男生，进女生宿舍不方便。建议：
1. 找女生朋友帮忙（付100元，扫2栋楼）
2. 或者只扫男生宿舍楼（效果会打折）


## 💰 第三部分：变现路径规划

### 短期变现（1-3个月，轻量级）

#### 方案1：校园餐厅广告（最推荐）

**逻辑：** 你有流量，餐厅需要客源（堂食/外卖）；以“结果页曝光+优惠引导”带客。

**落地步骤（两周内）**
1. 数据积累（第1周）：
   - 统计 Top10 被抽中餐厅、类目占比、使用高峰时段；
   - 准备“使用数据截图”（真实数据/示例数据占位，后续替换）。
2. 准备“一页纸”（PDF/图片）：
   - 项目介绍（1句话价值）/用户画像&规模/使用数据；
   - 投放位示意图（结果页/分享页/底部Banner）；
   - 合作套餐与价格/有效性衡量方式/联系方式&二维码。
3. 价卡与套餐（建议）：
   - Basic：结果页“推荐商家”标签+店铺卡片露出，300元/月；
   - Plus：Basic + 结果页优惠券弹窗（每人1次），500元/月；
   - Pro：Plus + 分享页露出 + 店内台卡二维码，800元/月。
4. 目标清单：优先校门口3-5家高频店（奶茶、简餐、麻辣烫、米线）。
5. 约谈话术（开场30秒）：
   - “我是本校学生，做了一个决定吃什么的小工具，上周有XX位同学使用，能帮附近商家导流。想给您做个试投，风险很低，今天即可上线。”
6. 异议处理（常见4问）：

## 📊 第四部分：数据监测与优化

### 追踪规范：短链/UTM/二维码
- 链路：落地页 → 抽取 → 分享/下单 → 留存；为每个渠道生成独立短链与二维码。
- 短链：每渠道单独短链（suo.im/dwz.cn），命名“w1_渠道_创意_日期”。
- UTM参数规范：
  - utm_source：canteen（食堂地推）/biaobai（表白墙）/qzone/kol/dorm；
  - utm_medium：offline/social/kol/qr；
  - utm_campaign：w1_launch；
  - utm_content：copyA/copyB（文案版本）；
  - utm_term：可选（如 dorm-3-414）。
- 链接示例：
  - https://yourdomain.com/?utm_source=biaobai&utm_medium=social&utm_campaign=w1_launch&utm_content=storyA&utm_term=2025w45
- 二维码：为每个渠道单独生成并贴“渠道名”小标签，避免混淆。
- 事件口径对齐PRD：app_launch/page_view/mode_selected/draw_clicked/draw_result/draw_again/share_clicked；
- 指标口径：
  - UV（按天去重）；有效用户（当周≥1次抽取）；
  - CAC=当日花费/当日新增；K因子=（分享点击UV/使用人数）。

### 关键指标追踪（每天必看）

| 指标 | 定义 | 目标值 | 获取方式 |
|------|------|--------|---------|
| **UV（独立访客）** | 每天有多少人打开 | Day1:10 → Day7:50+ | Google Analytics |
| **转化率** | 打开后实际使用比例 | >60% | localStorage统计 |
| **留存率** | 今天用的人明天还会来吗 | Day2:40% / Day7:20% | 用户ID追踪 |
| **分享率** | 使用后分享给朋友的比例 | >10% | 分享按钮点击统计 |
| **活跃时段** | 什么时候用的人最多 | 11:30-12:30 / 17:30-18:30 | 时间戳统计 |

### 数据追踪代码（快速接入）

在 `src/pages/Index.tsx` 中加入：

```typescript
// 简单的数据统计（无需后端）
useEffect(() => {
  // 记录访问
  const visits = Number(localStorage.getItem('total_visits') || 0) + 1;
  localStorage.setItem('total_visits', visits.toString());
  
  // 记录今日UV
  const today = new Date().toDateString();
  const lastVisit = localStorage.getItem('last_visit_date');
  if (lastVisit !== today) {
    localStorage.setItem('last_visit_date', today);
    const dailyUV = Number(localStorage.getItem('daily_uv') || 0) + 1;
    localStorage.setItem('daily_uv', dailyUV.toString());
  }
}, []);
```

## ⚠️ 第六部分：避坑指南

### 常见错误与解决方案

#### 错误1：产品没做好就开始推广

**表现：** 用户打开速度慢、体验差、bug多
**后果：** 推一个走一个，口碑崩盘
**解决：** 前2天务必先让种子用户测试，修复明显bug

#### 错误2：盲目大规模推广

**表现：** 第一天就去所有群发广告
**后果：** 被当成垃圾广告，被踢出群，后续推广受阻
**解决：** 先小范围验证（10-30人），确认产品好用再扩散

#### 错误3：没有闭环数据统计

**表现：** 不知道多少人用了、怎么来的、为啥走的
**后果：** 无法优化，瞎推广
**解决：** Day1就接入基础统计（localStorage或Google Analytics）

#### 错误4：过度承诺无法兑现

**表现：** 承诺"分享送奶茶"，结果500元预算不够
**后果：** 用户失望，品牌受损
**解决：** 福利设置"前XX名"限制，成本可控

#### 错误5：忽视女生用户心理

**表现：** 推广话术太直男、太功能性
**后果：** 女生（主要用户）不感兴趣
**解决：** 多用感性词汇（"救命""绝了""姐妹们"），少用技术术语

#### 错误6：没有建立私域流量

**表现：** 用户来了就走，没留联系方式
**后果：** 无法二次触达，无法推新功能
**解决：** 必须建微信群/公众号，地推时留微信

#### 错误7：变现太着急

**表现：** 还没几个用户就开始打广告、收费
**后果：** 用户体验差，留不住人
**解决：** 前1-2个月专注用户增长，1000+用户后再谈变现

## 🎯 第十部分：成功标准与退出机制

### 成功标准（Week 1）

**最低标准（及格线）：**
- ✅ 50个真实用户
- ✅ 日活>10人
- ✅ 至少1个用户主动分享

**良好标准（优秀线）：**
- ✅ 80个真实用户
- ✅ 日活>30人

- ✅ 有餐厅主动找你谈广告

**卓越标准（理想线）：**
- ✅ 100+真实用户
- ✅ 日活>50人
- ✅ 次日留存率>50%
- ✅ 出现自发传播（表白墙有人主动推荐）
- ✅ 谈下第一笔收入

### 退出机制（避免无效消耗）

**场景1：Week1结束<30人**
- 原因分析：产品问题 or 推广策略问题？
- 决策：如果产品问题，暂停推广，优化产品
- 决策：如果推广策略问题，调整话术/渠道

**场景2：用户来了就走（留存率<10%）**
- 原因：产品不好用 or 需求伪需求？
- 决策：深度访谈5-10个用户，找到真正问题
- 决策：如果改不了，考虑转型或放弃

**场景3：1个月后仍无收入**
- 原因：用户量不够 or 变现模式有问题？
- 决策：如果用户量<500，继续增长
- 决策：如果用户量>500但无收入，调整变现策略

**止损线：**
- 投入时间>50小时，用户<100人 → 考虑暂停
- 投入金钱>1000元，无任何收入 → 考虑暂停
- 持续3个月无增长 → 考虑转型或放弃

## ❓ FAQ 与异议处理

### 面向学生
- Q：这是广告吗，会不会很烦？
  - A：不是刷屏广告，只在结果页推荐校内真实餐厅/优惠，且可关闭；

  - A：建议使用校园Wi-Fi/4G；若仍卡可反馈截图，我们会当天修复。
